# cli-consultants v1.2.0 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add Plan-subagent as optional third consultant channel and mandate an agent-execution question on final-pass prompts; strictly additive (no breaking changes vs v1.1.0).

**Architecture:** Plan-subagent uses Claude Code's `Agent` tool (`subagent_type=Plan`, `model=opus`) — no external CLI, no auth setup. It joins Codex and Gemini as a third independent reviewer with fresh context per call. Operationally it serves three new roles: (a) bonus voice during dual final-pass when both Codex and Gemini are configured; (b) fallback when Codex is rate-limited or Gemini is drifting; (c) compensating voice in Codex-only mode (was: single-Codex with `Gemini SKIPPED` disclosure; now: Codex + Plan-subagent quasi-dual with softer disclosure). The agent-execution question is a single mandatory line appended to all final-pass prompts that catches execution-readiness bugs (UnboundLocalError, deadlocks, undefined helpers) which architectural review systematically misses.

**Tech Stack:** Plain Markdown skill files + Bash wrapper scripts already in v1.1.0. No new code, no new CLI. Plugin distribution via existing marketplace `https://github.com/lissergey/team-cli-consultant.git`.

## Verification Strategy

This is a skill-content change, not a code change. Per the RED→GREEN methodology used in earlier iterations of this plugin, verification = dispatching general-purpose subagents into pressure/application scenarios that target the new behaviors:

- **Scenario V1 (Plan-subagent as Codex-rate-limit fallback):** subagent is told Codex returned 429 mid-task during a spec final-pass; should use Plan-subagent to fill the slot instead of blocking or proceeding silently.
- **Scenario V2 (Agent-execution question on final-pass):** subagent is told to compose a final-pass Gemini question; the rendered prompt must contain the mandatory agent-execution question verbatim.
- **Scenario V3 (Codex-only mode + Plan-subagent compensation):** subagent in a project with only `tools/ask_codex.sh` doing a final-pass; report must contain `Gemini SKIPPED; Plan-subagent compensating as second voice` (not the old `Gemini SKIPPED ... single-Codex` line).

## Execution Handoff

Plan complete and saved to `docs/plans/2026-05-13-v1.2.0-plan-subagent-and-agent-execution.md`. Two execution options:

**1. Subagent-Driven (recommended)** — fresh subagent per task, review between tasks, fast iteration. Best when each task is independent enough to give a subagent its own context.

**2. Inline Execution** — run tasks in this session via the executing-plans skill, batch with checkpoints. Best given the tight cross-references between tasks (all touching the same SKILL.md repeatedly with sequential edits, where a fresh subagent per task would re-read the whole file every time).

For this plan, **inline execution is the better fit** — most tasks are sequential edits to two files (`using-cli-consultants/SKILL.md` and `CLAUDE_SECTION.md`), and the cost of re-reading them each subagent dispatch outweighs the parallelism benefit.
